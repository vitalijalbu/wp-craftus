# AUDIT COMPLETO — sage-theme

## Obiettivo
Tema corporate + e-commerce livello Shopify, basato su Sage 11, Tailwind v4, Alpine.js, WooCommerce. Il tema funziona ma ha **problemi architetturali**, **incoerenze nel design system**, e **funzionalità mancanti**.

---

## 1. DESIGN SYSTEM — theme.json + Tailwind (CRITICO)

### 1A. Palette colori: naming e slug sbagliati

| Problema | Dettaglio |
|----------|-----------|
| **Mancano slug standard WP** | `primary`, `secondary`, `tertiary`, `contrast`, `base` — servono per i blocchi Gutenberg |
| **"gold" = blu** | `"slug": "gold", "color": "#0074C7"` — confonde chi sviluppa, i template usano `text-gold` ma è blu |
| **No colori di stato** | Manca `success`, `warning`, `error` per form validation e notice WC |
| **`--color-muted` conflitto** | In theme.json = `#6b6b6b` (grigio scuro), in @theme di app.css = `#fafafa` (quasi bianco!) |

### 1B. Font: Inter etichettato "serif" ma è sans-serif

In theme.json:
```json
{ "fontFamily": "'Inter', system-ui, sans-serif", "slug": "serif" }
```
Inter è un **sans-serif**. Tutto il tema usa `font-serif` per heading ma ottiene Inter (sans), e `font-sans` per body ottenendo Poppins. Il risultato visivo funziona ma la semantica è invertita — confonde chiunque lavori sul tema.

### 1C. Custom properties vuote

Le sezioni `custom.spacing`, `custom.typography.font-size`, `custom.typography.line-height` in theme.json sono **tutte vuote** `{}`. Dovrebbero contenere token riutilizzabili come `header-height`, `section-padding`, `line-height-tight/normal/relaxed`, ecc.

### 1D. Font sizes non fluidi

Solo `hero` usa `clamp()`. Tutti gli altri 9 size sono statici. Per un tema luxury responsivo servono font-size fluidi almeno da `3xl` in su.

## 2. CSS — app.css (2.469 righe, problemi strutturali)

### 2A. Duplicazione massiva

| Classe | Proprietà duplicate | Quante volte |
|--------|-------------------|-------------|
| `font-family: var(--font-sans)` | In ogni `.btn-*`, `.nav-*`, `.contact-*`, `.badge`, ecc. | ~30 volte |
| `font-size: 0.8125rem` | Ripetuto ovunque per "small uppercase" | ~15 volte |
| `letter-spacing: 0.1em` | Bottoni, badge, label, tab | ~20 volte |
| `transition: ... 250ms ease` | Ogni componente lo ridefinisce | ~25 volte |

**Soluzione:** creare token CSS custom per `--duration-fast`, `--tracking-button`, `--text-button-size`, ecc.

### 2B. Stili WooCommerce con `!important` ovunque

In app.css ci sono **18 dichiarazioni `!important`** solo per i bottoni WC. Questo indica che la specificità non è gestita bene — servono layer CSS (`@layer`) per separare base/components/overrides.

### 2C. Componenti CSS non serviti da Tailwind

Ci sono ~40 classi BEM custom (`.product-card__*`, `.testimonial-card__*`, `.faq-item__*`, ecc.) che duplicano cose che Tailwind può fare con utility classes. Non è necessariamente un errore — è un pattern valido — ma manca coerenza: **alcune sezioni usano solo utility Tailwind, altre solo CSS custom, altre un mix**.

### 2D. Due sistemi di navigazione paralleli

Le classi `.site-header .nav-primary` (CSS custom in app.css) e il codice Alpine in header.blade.php con classi `.nav-link-t` gestiscono la stessa navigazione con due pattern diversi. Solo `.nav-link-t` è effettivamente usato dall'HTML.

## 3. TEMPLATE BLADE — Architettura

### 3A. front-page.blade.php è vuota

front-page.blade.php fa solo `the_content()`. Il Composer `FrontPage.php` inietta `$heroData` e `$storyData` dal Customizer ma **nessun template li usa**. Risultato: **9 impostazioni del Customizer (Homepage) sono orfane**.

### 3B. Partials inutilizzati

| File | Stato |
|------|-------|
| partials/content-page.blade.php | Mai incluso da nessun template |
| partials/content-search.blade.php | Mai incluso, search usa `content.blade.php` |
| partials/page-header.blade.php | Registrato nel Composer ma usato raramente |

### 3C. Container inconsistente

Il pattern dovrebbe essere `max-w-360 mx-auto px-6 lg:px-10` ma non è centralizzato. Ogni section/partial lo ridefinisce. Dovrebbe essere un componente Blade o una utility Tailwind custom `@apply container-luxury`.

### 3D. WooCommerce templates incompletI

| Template | Stato |
|----------|-------|
| Loop prodotti (archive) | ✅ Override via content-product.php → product-card.blade.php |
| Singolo prodotto | ⚠️ Override PHP diretto (`woocommerce/single-product/`), non Blade |
| Carrello | ⚠️ Usa solo `the_content()` (fix recente), nessun template custom |
| Checkout | ❌ Nessun override |
| Account | ❌ Nessun override, CSS base applicato |
| Email | ❌ Nessun override |

## 4. JAVASCRIPT — Alpine.js + GSAP

### 4A. Cleanup mancante

Gli event listener scroll in `siteHeader` Alpine component (app.js) non vengono mai rimossi. In una SPA-like navigation (se mai implementata) causerebbe memory leak.

### 4B. Wishlist: selettori sbagliati (parzialmente fixato)

Il wishlist.js cercava `.wishlist-btn` ma le card usano `.product-card__wishlist`. Fix applicato nell'ultimo commit, ma il custom element `<wishlist-products>` ha ancora un `fetch()` verso `${theme.routes.root_url}search` — un endpoint che **non esiste** in questo tema WordPress (è un pattern Shopify).

### 4C. Locomotive Scroll caricato ma non usato

Il modulo locomotive-scroll.js è importato ma non attivato su nessun elemento nel DOM. Overhead inutile.
