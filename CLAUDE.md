## Coding rules

- **Blade templates**: use `{!! !!}` only for escaped WP output (`get_the_title()` etc.) or trusted HTML. Never raw user input.
- **PHP**: namespace `App\`, PSR-12, strict types where practical. No inline SQL — always WP_Query / $wpdb->prepare().
- **JS**: ES modules, no jQuery except where WC requires it (fragment refresh). Alpine components live in `resources/js/app.js`.
- **CSS**: Tailwind utility-first. Design tokens are in `@theme {}` in `app.css`. Never add inline `style=""` for design — use Tailwind classes.
- **WooCommerce templates**: override in `woocommerce/` (PHP only). Use Blade product-card partial via `\Roots\view()`.
- **Accessibility**: every interactive element needs a visible label or `aria-label`. Carousels use full ARIA carousel pattern. Filter groups use `role="group"` + `aria-pressed`.
- **Security**: always `esc_url()`, `esc_html()`, `esc_attr()`, `wp_kses_post()` on output. `sanitize_*` on input. No `$_GET`/`$_POST` direct access in templates.
